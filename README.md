# Data-Society-Group---Power-BI-Dashboard
Sales Overview Dashboard


Objective:
Build an executive-ready dashboard using the dataset provided with the following steps
1.	Model the data appropriately
2.	Create meaningful DAX measures
3.	Design a clean, insight-driven report
4.	Demonstrate clear thinking around data structure and business storytelling

Data Context

The datasets represent transactional sales data for a company selling products across multiple regions and customer segments over a two-year period (2023–2024).
There are four tables:
1.	Sales: Transaction-level order data including order date, customer, product, region, quantity, unit price, and discount.
2.	Customers: Customer details including segment (Enterprise, SMB, Consumer) and signup date.
3.	Products: Product hierarchy including category and subcategory.
4.	Calendar: A full date table covering 2023 - 2024 for time-based analysis.

The data is clean and ready for modeling. Relationships are not pre-defined

Create a Power BI report that:

Data Modeling
1.	Create appropriate relationships between tables
2.  Ensure a proper star schema structure
3.  Build and correctly utilize a usable Date table
4.  Avoid flattening everything into a single table
